Digital Twin–Driven Safe Exploration for Autonomous IoT Network Optimization

Jul – Sep 2026

Network Digital Twins (NDTs) are becoming the backbone of autonomous network management — the core of any NDT is a network model that turns topology, routing, and traffic state into accurate KPI predictions. This track asks a direct question: can replacing TwinNet's Message-Passing GNN backbone with GAT, a Graph Transformer, or Graph Mamba improve prediction accuracy, generalization, or inference efficiency?

Network Digital Twins Graph Neural Networks Graph Mamba / SSMs IoT Scalability TwinNet
Why it matters

TwinNet (Ferriol-Galmés et al., 2022) is the leading open-source NDT for QoS-aware optimization, generalizing well across unseen topologies with a Message-Passing GNN. Since then, GAT, Graph Transformers, and Graph Mamba have each claimed to fix a limitation of standard message passing — but none had been systematically evaluated on a network digital twin task before this study.

Research questions

RQ1Does architecture choice affect prediction accuracy on TwinNet's benchmark scenarios?Justifies replacing the original backbone with a better-performing alternative.

MAPE, unseen-topology generalization, inference latency, training time, and memory.

IoT-scale extension

Beyond TwinNet's original dataset, the study generates a Mininet-based IoT topology set (10–50 nodes, MQTT/CoAP traffic from CICIoT2023) to test behavior specifically at IoT scale.

Future direction — post-conference

Graph Hydra: dynamic expert selection over specialized graph heads

TwinNet actually carries four different kinds of signal at once — topology, traffic, routing, and queue state — each suited to a different architecture. Graph Hydra's proposal: give each stream its own specialist head (structural GNN, temporal Mamba, path-aware Transformer, local GAT), then use a lightweight controller to decide which heads matter for the current network state, rather than always running all of them. If the comparative study confirms which architecture wins per input type, Graph Hydra becomes the natural Phase 2.

Roadmap

From comparative study to standalone architecture.

Phase 1 · Current proposal

Complete the comparative study
